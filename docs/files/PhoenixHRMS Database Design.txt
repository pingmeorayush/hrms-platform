1. Purpose
This document defines the database architecture of PhoenixHRMS.
Objectives:
	•	Data consistency
	•	Scalability
	•	Multi-tenancy
	•	Auditability
	•	Reporting support
	•	Compliance support

2. Database Technology
Primary Database:
PostgreSQL 17+

Reasons:
	•	ACID Compliance
	•	Strong indexing
	•	JSON support
	•	Full text search
	•	Partitioning support
	•	Enterprise reliability

3. Design Principles

DB-001
UUID Primary Keys
Every table uses UUID.
Example:
id UUID PRIMARY KEY
Benefits:
	•	Security
	•	Distributed systems readiness
	•	Future microservices support

DB-002
Multi-Tenant By Design
Every business table contains:
company_id UUID

DB-003
Soft Deletes
Important business tables use:
deleted_at TIMESTAMP

DB-004
Auditability
Critical entities maintain history.

DB-005
Normalization First
Target:
3NF
Exceptions allowed for reporting optimization.

Conclusion
The PhoenixHRMS database architecture is designed around PostgreSQL, multi-tenancy, auditability, scalability, and future ERP expansion.
The schema follows strict normalization principles while maintaining flexibility for reporting, analytics, payroll processing, and AI-driven insights.
All future migrations, models, repositories, APIs, and reporting layers must align with the structures and principles defined in this document.
