1. Purpose
This document defines all entity relationships within PhoenixHRMS.
Objectives:
	•	Define entity ownership
	•	Define cardinality
	•	Define aggregate boundaries
	•	Define foreign key relationships
	•	Define cascading rules
	•	Support implementation

2. ERD Standards

Ownership Rules

Conclusion
The PhoenixHRMS ERD architecture establishes clear ownership, relationships, aggregate boundaries, and data integrity rules across all domains.
The Employee entity serves as the primary business aggregate, while Company acts as the tenant root aggregate.
These relationships provide the foundation for migrations, repositories, APIs, services, reporting, auditing, and future ERP expansion.
