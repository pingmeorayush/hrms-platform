## How To Use These Docs

- Use each sprint file as the planning baseline for engineering, product, QA, and platform work.
- Treat the `Primary Backlog IDs` section as the bridge to implementation tickets.
- Update the `Risks and Open Questions` section at sprint planning and sprint review.
- Keep detailed feature and API design in module or architecture docs, not in the sprint plan itself.
